## v1.5.0 更新内容

### 🐛 P0 修复 AI 不会用 gamemode 命令（核心修复）

**根因**：`CommandExecutor` 已实现 gamemode 等 21 个命令，但 `config.yml` 的 system prompt 完全没告知 LLM，导致 AI 不知道自己能直接改 gamemode，只能"寻找方块放置命令块"。

- 补全 21 个缺失命令文档（gamemode/kill/heal/feed/fly/ignite/strike/explode/spawnmob/xp/clearinv/duplicate/openinv/rename/ride/carry/home/top/combo/emote）
- NPC 实体设置 OP（`entity.setOp(true)`），能通过 `Bukkit.dispatchCommand` 执行服务器命令
- 明确告知 LLM："你可以直接修改自己的状态（gamemode/heal/feed/fly），无需 OP 权限"

### ✨ P1 命令文档自动同步 + 执行结果回流

- 新建 `@AICommand` 注解，60 个命令方法加注解
- `CommandExecutor.getCommandDocs()` 反射扫描生成文档
- `ConversationManager` 动态注入命令清单，删除 config.yml 硬编码
- `ExecutionResult` 类，execute 返回执行结果
- `AIPlayer.lastCommandResult` 存储上一轮结果
- 下一轮 LLM 调用注入"你上一轮 [COMMAND:xxx] 失败：原因"，AI 能学习纠正

### ✨ P2 反派 AI 人设 + 目标驱动

- `Personality` 新增 VILLAIN/CONQUEROR/MANIPULATOR/STRATEGIST 反派人格
- `/aip villain on/off` 一键开启反派模式
- `GoalManager` + `Goal` 类，`/aip goal add/list/complete/progress`
- 目标注入 system prompt 驱动 AI 行动
- `ai.autonomous` 默认 `true`，AI 无人 @ 时也会基于目标自主行动
- 自主决策 prompt 改为目标驱动："基于你的目标、进度、最近事件，决定下一步战略动作"

### ✨ P3 长期记忆 + 玩家档案 + 策略引擎

- `LongTermMemory` 突破 20 条对话限制，MAX_SIZE=200
- `MemoryRecord`：DEATH/ATTACK/DECEIVE/CLAIM/ALLIANCE/DISCOVERY 事件类型
- `PlayerProfileManager` 为每个真实玩家维护档案（威胁等级/攻击历史/装备/关系值）
- `StrategyEngine`：fake_friendly/backstab/trap/feint 预设策略
- `[COMMAND:strategy fake_friendly Steve]` 调用策略

### ✨ P4 服务器控制 + 多 AI 协同

- `op/deop/ban/kick/tp_all/gamemode_player` 服务器控制命令（受 allow-op-commands 控制）
- `TeamManager` 新增作战角色（DECOY/ASSAULT/SUPPORT/SCOUT）和协同目标
- `TaskManager` 新增 siege/sabotage/infiltrate 任务类型
- `/aip memory show` / `/aip profile show` 子命令

### 📊 规模

- 新增 9 个类：`AICommand`/`ExecutionResult`/`Goal`/`GoalManager`/`LongTermMemory`/`MemoryRecord`/`PlayerProfile`/`PlayerProfileManager`/`StrategyEngine`
- 30 个文件变更，+1724 行 / -159 行
- jar 大小 160KB
